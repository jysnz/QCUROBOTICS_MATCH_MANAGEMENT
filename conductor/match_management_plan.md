# Tournament Match Management System Plan

## Objective
Design and implement a comprehensive Tournament Match Management System for a robotics competition platform, leveraging Supabase for robust database-driven logic and Flutter for a responsive, modern UI.

## Background & Motivation
The platform needs a centralized dashboard to create and manage tournaments. The core challenges involve scheduling a fair tournament where each team plays exactly 11 matches, and maintaining real-time, accurate rankings based on match outcomes. By pushing the heavy lifting (scheduling algorithms and ranking recalculations) to the database layer, we ensure data integrity, prevent race conditions, and enable seamless real-time updates for all clients.

## Scope & Impact
This feature introduces significant database schema additions and complex UI screens. It impacts:
- **Database:** New tables (`tournaments`, `tournament_teams`, `matches`, `rankings`), RPCs for match generation, and triggers for ranking updates.
- **Backend Services:** Dart models and Supabase service wrappers to interface with the new schema.
- **Frontend UI:** Dashboard page, Tournament Creation flow, Tournament Details page, Match Management modals, and real-time Ranking tables.

## Proposed Solution: Database-Driven Architecture

### 1. Database Schema
- **`tournaments`**: `id`, `name`, `date`, `status` (Ongoing/Completed), `created_at`.
- **`teams`**: (Assumed existing) `id`, `team_name`.
- **`tournament_teams`**: Join table mapping `tournament_id` to `team_id`.
- **`matches`**: `id`, `tournament_id`, `match_number`, `red_team_id`, `blue_team_id`, `red_score`, `blue_score`, `status` (Pending/Completed), `created_at`.
- **`rankings`**: `tournament_id`, `team_id`, `wins`, `losses`, `ties`, `matches_played`, `total_points_scored`, `total_points_conceded`, `ranking_points`.

### 2. Database Logic (RPCs and Triggers)
- **RPC `generate_matches(t_id uuid)`**: A PostgreSQL function that takes the teams in a tournament and generates a schedule where every team plays exactly 11 matches, optimizing for unique pairings and balanced color assignments.
- **Trigger `update_rankings_on_match_completion`**: A database trigger on the `matches` table. When a match's status changes to 'Completed', it automatically recalculates the wins, losses, ties, points, and ranking points for both the Red and Blue teams in the `rankings` table.

### 3. Flutter Architecture & UI
- **Design System:** Implement the `Soft UI Evolution` pattern generated by UI/UX Pro Max, utilizing Fira Code/Sans typography, modern technical coloring (Slate/Green), and responsive layouts.
- **Models:** `Tournament`, `Match`, `Ranking`, `Team`.
- **Pages:**
  - `DashboardPage`: Overview of recent tournaments and the "Start Tournament" CTA.
  - `TournamentDetailsPage`: Tabbed or sectioned view containing Information, Match Schedule, and Live Rankings.
- **Components:**
  - `CreateTournamentModal`: Form to input details and select teams dynamically.
  - `StartMatchModal`: Form to input scores for Red and Blue teams with validation.
  - `RankingTable`: Real-time Supabase stream listener displaying the updated leaderboard.

## Implementation Steps

### Phase 1: Database Setup
1. Define the SQL schema for the new tables.
2. Write the PL/pgSQL function (`generate_matches`) for the 11-match scheduling algorithm.
3. Write the trigger function (`update_rankings_on_match_completion`) for live scoring.
4. Execute the SQL against the Supabase instance (or provide it to the user to run).

### Phase 2: Core Data Models & Services
1. Create Dart models for `Tournament`, `Match`, and `Ranking`.
2. Implement `TournamentService` handling Supabase calls (fetch tournaments, create tournament, call RPC to generate matches, submit match score).

### Phase 3: Dashboard & Tournament Creation
1. Build `DashboardPage` listing recent tournaments.
2. Build `CreateTournamentModal` allowing manual team selection.
3. Wire up tournament creation and invoke the match generation RPC.

### Phase 4: Tournament Details & Match Management
1. Build `TournamentDetailsPage`.
2. Implement the Match Schedule table.
3. Build the `StartMatchModal` for score entry and submission.
4. Add logic to check for tournament completion (all matches status = 'Completed').

### Phase 5: Real-time Rankings
1. Implement the `RankingTable` widget.
2. Setup Supabase Realtime subscription on the `rankings` table to update the UI instantly when scores are submitted.

## Verification
- Create a test tournament with 4-8 teams.
- Verify the RPC generates matches such that each team plays 11 times.
- Submit a match score and verify the trigger correctly updates both teams' rankings.
- Ensure the UI reflects the ranking changes in real-time without a manual refresh.
- Complete all matches and verify the tournament transitions to a 'Completed' state.
